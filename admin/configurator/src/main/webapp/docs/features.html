Feature Reference

Header controls

Target
Which scope's JSON file you are editing. See Target Scopes.
Schema
Which deployed BLADE app's settings you are editing. See Schema Discovery.
WebSocket status
Live indicator: green means the editor is connected to the WAR's WebSocket and will see remote file-system changes (e.g. another operator saving). Yellow during reconnect.
Dirty indicator
The orange dot + "Unsaved Changes" appears any time form state diverges from the last loaded JSON. Save or Load clears it.
Theme
Light, Dark, Midnight Blue, or Warm Sand. Theme is per-browser (localStorage), not per-user or per-deployment.

Tabs

Form Editor

The primary editing surface. The form is generated from the schema: every property becomes a labeled field, every nested object becomes a collapsible section, every array becomes a repeatable group. Field descriptions come from @JsonPropertyDescription annotations on the Java class.

Sub-features:

  • Filter — search box narrows visible fields to those whose label or path matches.
  • Breadcrumb — when a field is focused, shows its full dotted path. Click to copy.
  • Auto-save draft — uncommitted form state is mirrored to localStorage every few seconds. If you close the tab and reopen later, the Draft banner offers to restore.
  • Validation summary — schema-driven validation errors are listed at the top with click-to-jump links.

JSON Editor

Raw JSON view of the same document, in an Ace editor with JSON syntax + bracket matching + format-on-button. Useful for surgical edits that are awkward in the form (deleting a whole sub-tree, pasting in a known-good snippet).

Format JSON reformats with two-space indentation. Validate runs both syntactic JSON parsing and schema validation. Save from this tab persists the same way as Form Editor.

JSON Schema

Read-only view of the schema itself. Same Ace editor. Useful for tracing why a field validates a certain way or what enum values are permitted.

Files

Free-form text editor for the auxiliary template files that live under $DOMAIN_HOME/_templates/. These are not JSON config — they are body templates for REST connectors, SQL for JDBC connectors, LDAP filter strings, etc. Variable substitution at runtime uses the BLADE ${var} syntax.

The Files tab has a left rail listing every template file and an Ace editor on the right with mode chosen from the file extension.

Toolbar menus (Form Editor tab)

File

New from Template…
Pick a starter config from the template library. Loaded into the form but not saved until you press Save.
Load Sample
Reloads the framework-generated SAMPLE for this app. Useful as a known-good baseline.
Import…
Choose a JSON file from your local disk. Replaces the form contents (with confirmation).
Export
Downloads the current form state as a JSON file.
Reset Form
Reverts unsaved changes back to the last loaded JSON.

History

Version History
Lists every saved revision for the current target + schema, with timestamps and operator info. Click to view; "Restore" promotes an old revision to current.
Diff…
Side-by-side diff of the current unsaved state against the last-loaded server copy. Useful before pressing Save to confirm you're shipping what you think you're shipping.
Clone to Another Target
Copies the current form state to a different target scope (Domain → Cluster, Cluster A → Cluster B, etc.). Convenient for promoting a tested cluster override to domain.

View

Expand All / Collapse All
Open/close every collapsible section in the form.
Collapse Empty
Collapses only sections whose fields are all null/default. Useful for hiding noise.
Inspector
Toggles the right-side panel showing focused-field schema + variable palette.
View Mode
Toggle between expanded (default) and compact form density.

Inspector side panel

Toggled from View → Inspector. When a field has focus, the panel shows that field's schema fragment (type, description, validation rules, enum values) and the list of available ${variables} defined in the current document (selectors with an id). Click a variable to insert it at the cursor.

Modals

Template Library
Opened by File → New from Template. Picker of starter configs.
Diff
Per-path change list with old → new value.
Regex Tester
Opened by the test button next to any field bound to RegexSelector.pattern. Paste sample text, see whether the regex matches.
Version History
Time-ordered revisions with diff and restore.
Clone Configuration
Pick a destination target, confirm, and the current form state is written there.

Embed mode

Loading the editor with ?embed=1 hides the brand topbar and the page title heading, and tightens top padding. The portal uses this when iframing the editor so the operator doesn't see a chrome-on-chrome stack.
