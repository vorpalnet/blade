Welcome

The BLADE Configurator is a schema-driven editor for every BLADE service's JSON configuration. It discovers available settings from the live OCCAS domain, renders a form from each schema, and round-trips edits through the file-system to the running application's MBean — without an application restart.

Reading order

If you are new to the Configurator, read in this order:

  1. Overview — what the editor does and how it fits into the BLADE config story.
  2. Target Scopes — Domain / Cluster / Server precedence and when to use each.
  3. Schema Discovery — how the dropdown gets populated.
  4. Feature Reference — per-tab and per-menu reference for the editor itself.

At a glance

PatternJSON-Schema-driven form editor with live JMX round-trip
Edits land in$DOMAIN_HOME/config/custom/vorpal/[scope]/<app>.json
Schemas come fromEvery BLADE app's SettingsManager auto-registers a vorpal.blade:Type=Configuration MBean exposing a generated schema
Reload triggerFile system watcher (in this WAR's ConfigurationMonitor) detects changes and pushes JSON to the matching SettingsMXBean via openForWrite / write / reload
Live stateWebSocket on the page reflects file-system + MBean state; the dirty indicator shows unsaved local edits
VersioningEach Save writes a timestamped revision; View → History opens the version list with full diff
Quick action

The Editor tab opens the form editor. Pick a target scope from the first dropdown, then a schema. Changes are validated against the JSON Schema and version-tracked; nothing reaches the cluster until you press Publish.
