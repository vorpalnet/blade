Sections Reference

JVM

Per-server JVM arguments — what the JVM starts with on the next boot. All restart-required.

KnobTypical valueEffect
-Xmssame as -XmxInitial heap size; pinning to max avoids reallocation pauses.
-Xmx4–16 GB for engine tierMaximum heap. Sized for expected concurrent sessions × per-session overhead.
GC algorithm-XX:+UseG1GC for mostPicks the GC strategy. G1 for low-pause requirements; ZGC for very large heaps.
GC log-Xlog:gc*:file=...Where GC events get written; essential for any sizing investigation.

SIP Timer

The SIP timer table per RFC 3261 §A. Tuning these affects retransmission cadence and dialog teardown. Runtime-tunable.

TimerDefaultWhat it controls
T1500msRound-trip estimate. Floor for retransmission intervals.
T24sMaximum retransmission interval for non-INVITE.
T45sMaximum duration a message persists in the network.
Timer F32s (64·T1)Non-INVITE transaction timeout.

Increase T1 on lossy / high-latency networks; decrease only if you've measured and know you can.

Work Manager

How WLS schedules incoming work across threads. Runtime-tunable.

KnobTypicalEffect
Min threads5Lower bound on the work-manager pool.
Max threads50–200Upper bound. Set to roughly 4× CPU count for I/O-bound workloads.
Capacity constraint50000Max queue depth before requests are rejected. Backpressure setting.
Fair-share class50Relative scheduling weight when multiple work-managers compete.

Node Manager

How the AdminServer talks to the per-host Node Manager for start/stop. Restart-required for listen port; debug toggle is runtime.

Listen address
The IP / hostname Node Manager listens on. Must match the AdminServer's NodeManager machine binding.
Listen port
Typically 5556 (SSL).
Debug
Verbose NodeManager logging; useful when start/stop sequences are misbehaving.
Log retention
How many historical NM logs to keep on disk.

Cluster

Cluster membership view (mostly read-only) plus the heartbeat knobs. Read-only members; heartbeat is runtime-tunable.

Members
Read-only list of servers in the cluster, including their listen address and current state.
Messaging mode
Unicast (preferred) or multicast. Unicast scales better and traverses routers; multicast needs a switched LAN.
Heartbeat interval
How often each member announces itself. Lower = faster failure detection but more network chatter.

SSO

Single-sign-on cookie configuration for the BLADE admin tier. Runtime-tunable.

Cookie name
Standard is BLADEADMINSESSION; every admin WAR shares this so a single login covers the whole admin tier.
Cookie path
Must be / for cross-WAR session sharing.
Cookie domain
Set only if the admin tier spans multiple subdomains.

Server Tuning

Per-server execute-queue and socket tuning. Mostly runtime; native-I/O toggle is restart-required.

Accept backlog
OS-level listen queue depth. Increase for high-burst clients (default 50 is low for production).
Socket receive/send buffer size
TCP buffer tuning. Defaults are usually fine; bump only after measuring.
Native I/O
Toggle WLS native socket muxer vs. pure-Java. Native wins for high concurrency; the toggle requires restart.
Always measure

Every "typical value" above is a starting point. Productionize tuning is workload-specific — measure before and after every change. If the change doesn't show on a graph, it didn't happen.
