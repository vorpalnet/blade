JDBC Connection Pools
Pool sizing. For a high-CPS deployment, set Initial = Min = Max so the pool is fully allocated at startup and never pays connection create/teardown latency under load. Min/Max resize on activate; Initial Capacity is restart-required.

SNMP Traps
Domain SNMP agent. BLADE services emit SIP-application traps from their logging config (set SNMP Trap Level per service in the Configurator). A trap only leaves the box when the agent below is Enabled, Send Automatic Traps is on, and at least one trap destination (your NMS host:port) is configured. Agent enable/port changes take effect when the agent restarts; destination edits apply on activate.

Overload Protection
Throttles new SIP work when the engine is overloaded (the classic <overload> element). Leave the policy blank to keep it unconfigured. Note: OCCAS also ships a separate, richer <overload-protection> framework that some domains scaffold instead — if your engine uses that, these fields are not the active control.
OCCAS Thread Tuning
WLSS Work Managers control how the SIP engine allocates threads to different processing functions. Fair Share is a relative weight (higher = more CPU). Min/Max Threads set hard bounds. Capacity is the max queued requests before the engine starts rejecting. Changes take effect on server restart.

Coherence
Coherence cluster networking — clustering mode (unicast / multicast), listen addresses and ports, TTL — from each Coherence system resource descriptor (config.xml). Restart-required: saved to the domain config; the engine tier must restart to pick it up.

Kernel Parameters
Read-only kernel tunables per node — sysctl values, transparent hugepages, CPU governor, process limits — read from /proc/sys, /sys, and /proc/self/limits on each node (no sudo). Guidance flags are SIP-oriented hints, not hard rules. Changing these is an OS task (Tier 2), not done here.
