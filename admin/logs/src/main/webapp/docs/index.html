Welcome

This handbook documents the BLADE Logs viewer — the admin app that lets you tail any log file on any cluster node from one page. The implementation is small but the mechanism is worth understanding: it federates across the domain via WebLogic's DomainRuntime MBeanServer and uses per-node MBeans to do the actual file read.

Reading order

  1. Overview — the architecture in one page.
  2. Cluster Discovery — how the server dropdown is populated.
  3. Reading Log Files — the slice/tail semantics and what offset / max bytes mean.

At a glance

TopologyOne viewer page (AdminServer) + one MBean per cluster node
DiscoveryJNDI lookup of java:comp/env/jmx/domainRuntime, walk DomainRuntimeServiceMBean.DomainConfiguration.Servers
Per-node MBeanvorpal.blade:Type=LogReader,Name=<serverName> — registered by every WAR via the framework's LogReaderRegistrar @WebListener
REST endpointsGET api/servers, GET api/servers/{name}/logs, GET api/servers/{name}/logs/{path:.+}
Read mechanismJMX invoke of LogReader.readSlice(serverName, relPath, offset, maxBytes) on the named-server MBean (federated via DomainRuntime)
Quick action

Open the Viewer, pick a server, pick a log file, and adjust tail bytes. No SSH, no per-host paths to remember.
