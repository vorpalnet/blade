Troubleshooting

SymptomLikely causeWhat to do
List request returns 404 The target service doesn't implement the CRUD endpoints, or the app name in the URL is wrong Confirm ?app=<name> matches a deployed service. Check that service's web.xml for the CRUD endpoint mapping.
List returns rows but the form is text-only The list response didn't include a schema This is the expected fallback. To get typed forms, the service must include the schema field in its response.
PUT succeeds but list doesn't show the new row List cache or pagination — the row exists but isn't in the visible page Refresh. If still missing, search for the row by primary key.
Preview returns 4xx The change is invalid per the service's rules Read the response body — services that implement preview should return a useful error message describing why.
Editor returns 401 on every request Session expired Re-login via the portal.
Always preview mutating ops

For table-shaped data, one wrong row can affect every call that matches. Use the dry-run preview for any production change — the few extra seconds are worth more than a recovery story.
