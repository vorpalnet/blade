Troubleshooting

Symptom-to-cause table

SymptomLikely causeFix
Stencil icons are broken (alt-text instead of glyphs) mxGraph stencil PNGs failed to load — usually a 404 on stencils/ or resources/ Verify those directories were packaged into the WAR (jar tf flow.war | grep stencils/). If missing, the war-plugin's packagingExcludes may be too aggressive.
Console error: mxResources: cannot get string for key … mxGraph i18n bundle didn't load. Harmless cosmetic — the editor falls back to the key as the label. Usually safe to ignore. If you want the right labels, ensure resources/editor.txt is reachable.
Save / Export silently does nothing Save servlet path mismatch — the editor POSTs to a relative URL that doesn't resolve to FsmarExportServlet Browser devtools → Network. Look for a 404 on the Export action. Confirm the WAR's web.xml has FsmarExportServlet mapped to the path the editor uses.
Editor steals focus from other windows (e.g. Confirm dialogs) jQuery UI window focus-trap quirk — a known mxGraph + jQuery UI interaction Click the canvas once to dismiss the trap, then operate the modal.
ClassNotFoundException during deploy / runtime Classloader conflict — usually prefer-web-inf-classes on while using the shared lib's prefer-application-packages Remove <prefer-web-inf-classes> from this WAR's weblogic.xml. See the project memory note weblogic-no-prefer-web-inf-classes.
Imported flow has all states in a vertical stack FSMAR XML was edited outside the editor — layout side-car got dropped Re-arrange in the canvas, Export. The layout side-car will be re-emitted.
Ingress / Egress clouds appear at unexpected positions on import They are synthesized on Import; their position is not stored in the descriptor Move them to wherever you want, Export, re-Import. The positions are sticky once exported via the layout side-car.
Active routing not changing after Export Engine-tier needs a reboot to pick up the new descriptor Roll the engine-tier servers per the project DEPLOYMENT.md.

Log signatures

FsmarImportServlet exception

org.vorpal.blade.applications.console.mxgraph.FsmarImportServlet
    java.io.IOException: cannot read fsmar descriptor

FSMAR JAR not where expected, or descriptor file inside it has the wrong name. Verify $DOMAIN_HOME/approuter/ contents and the JAR's internal layout.

FsmarExportServlet exception

org.vorpal.blade.applications.console.mxgraph.FsmarExportServlet
    org.xml.sax.SAXException: bad mxGraph payload

Editor sent malformed XML — usually a JS error mid-save corrupted the payload. Reload the editor, re-import, and try again.

Backup before Export

The editor overwrites the active descriptor on Export. Keep a copy of the prior XML before experimenting — recovery is easier than reverse- engineering a broken flow.
