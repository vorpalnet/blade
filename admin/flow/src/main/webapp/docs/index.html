Welcome

This handbook covers the Flow Editor, which is a visual front-end for FSMAR descriptors. If you have ever hand-edited an fsmar.xml file, this is the GUI equivalent — same source of truth, much friendlier authoring.

Reading order

If you are new to flow authoring, read in this order:

  1. Overview — what the editor produces and where it sits in OCCAS request handling.
  2. FSMAR Concepts — states, transitions, triggers, conditions, initial vs. non-initial requests.
  3. Editor Guide — the canvas, palette, inspectors, and the Tasks window.
  4. Import / Export Round-Trip — how the visual graph becomes the deployed XML and back again.

At a glance

ProducesFSMAR XML descriptor, shipped in fsmar.jar at $DOMAIN_HOME/approuter/
Canvas enginemxGraph (jgraphx 4.2.2)
BackendFsmarExportServlet (graph → XML) and FsmarImportServlet (XML → graph) in this WAR
ActivationFSMAR is loaded by OCCAS via the SPI javax.servlet.sip.ar.spi.SipApplicationRouterProvider; the engine tier reboots to pick up a new descriptor.
Triggers fire onInitial SIP requests only — see FSMAR Concepts for the full method list.
Quick action

The Editor tab opens the canvas with the last-imported flow loaded. Drag a stencil from the palette, double-click a state to edit, drag from state-edge to state-edge to add a transition. Press Export to round-trip changes back to the FSMAR XML.
