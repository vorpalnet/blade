Welcome

This handbook covers the Flow Editor, which is a visual front-end for FSMAR 3 configurations. If you have ever hand-edited an FSMAR JSON config file, this is the GUI equivalent — same source of truth, much friendlier authoring.

Reading order

If you are new to flow authoring, read in this order:

  1. Overview — what the editor produces and where it sits in OCCAS request handling.
  2. FSMAR Concepts — states, transitions, triggers, conditions, initial vs. non-initial requests.
  3. FSMAR 3 Tutorial — a hands-on, feature-by-feature tour with loadable example configs you run in the Route Simulator.
  4. Editor Guide — the canvas, palette, inspectors, and the Tasks window.
  5. Import / Export Round-Trip — how the visual graph becomes the deployed JSON and back again.

Triggers fire onInitial SIP requests only — see FSMAR Concepts for the full method list.

Route Simulator

The play-head toolbar button opens the Route Simulator — three tabs over one trace format:

  • Simulate — fill in a synthetic request (or paste a real INVITE from a trace) and run it through the diagram being edited. The routing path animates hop by hop on the canvas: values extracted, every transition shown FIRED / no match with its when, resolved routes. Override ${hour} / ${dayOfWeek} / ${hash100} to simulate Sunday 3 AM or a canary bucket, and mark applications undeployed to watch bypass, cycle-detection, and default-fallback behavior — all before deploying anything.
  • Replay — arm the engines' trace capture (per-call routing traces recorded by FSMAR 3 itself) and replay real production calls through the same animation.
  • Live — overlay per-transition hit counts from every engine on the diagram edges; the count is the label and edge thickness scales with traffic share.
Quick action

The Editor tab opens the canvas with the last-imported flow loaded. Drag a stencil from the palette, double-click a state to edit, drag from state-edge to state-edge to add a transition. Press Export to round-trip changes back to the FSMAR JSON — or the play-head button to simulate a call through it first.
