Editor Guide

Layout

The editor uses mxGraph (jgraphx). The viewport is divided into:

Canvas
Main drawing surface — grid background, panning with middle-mouse or right-drag, zoom with wheel + Ctrl.
Palette
Left rail with stencils (states, decision shapes, Ingress/Egress clouds). Drag onto the canvas to add.
Tasks window
Floating panel with filtered overviews — lists every state, every transition, and every condition in one scrollable list with click-to-jump.
Inspector
Side panel that updates with the selected cell — edit name, condition, trigger, and FSMAR-specific attributes there.

Common actions

Adding a state

  1. Drag a stencil from the palette onto the canvas. The default stencil is an application-invocation state.
  2. Double-click to edit the label. The label becomes the application name that ends up in the application chain.

Adding a transition

  1. Hover over a source state's edge — handles appear.
  2. Drag from a handle to the target state. A directed edge is drawn.
  3. Double-click the edge to set its label (typically the human-readable condition name).
  4. Open the Inspector to set the actual condition expression and (for Ingress-originating edges) the trigger method.

Editing a trigger

Triggers attach to the transition out of Ingress. Select that transition; in the Inspector, set the SIP method (INVITE, REGISTER, etc.) and any additional match criteria. The method dropdown is constrained to the methods FSMAR supports — see FSMAR Concepts.

Editing a condition

Conditions attach to any transition. Select the transition; in the Inspector, type or paste the condition expression. Syntax depends on FSMAR version. The Tasks window's condition list is the fastest way to audit every condition in the flow without clicking each edge.

Tasks window

The Tasks window is a flat overview of the otherwise deep graph. Useful for:

  • Finding orphan states (no incoming transitions).
  • Auditing every condition for a typo or stale predicate.
  • Jumping to a transition by name when you can't find it on the canvas.

Filter panel inside Tasks lets you scope by cell type (state / transition / condition) and by name regex.

Sub-pages

The editor uses three additional HTML pages as iframed inspectors when editing complex attributes:

state.html
Detailed state attribute editor — opened from a state's inspector when you need more form real estate.
transition.html
Detailed transition / condition editor.
config.html
Editor-wide settings (grid snap, autosave toggle, default styles).
Save vs. Export

The editor's Save action mirrors the canvas to localStorage so a reload restores your work-in-progress. Export is what writes the FSMAR XML back to the server-side servlet — that's the action that ships a new descriptor.
