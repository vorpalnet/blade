Welcome

The BLADE Watcher is a tiny backend service. It does one thing: watches $DOMAIN_HOME/config/custom/vorpal/ for file-system changes and pushes any modified JSON into the matching SettingsMXBean. Same mechanism the Configurator uses internally, packaged as a standalone WAR.

At a glance

UINone — landing + docs only
JavaWatcherStartup (@WebListener) starts a ConfigurationMonitor thread on context init
Watches$DOMAIN_HOME/config/custom/vorpal/** (recursive)
Action on changeMatch file → MBean by name + scope, push JSON via SettingsMXBean.openForWrite / write / close / reload
Why this exists

Historically the Watcher was the sole file → MBean pump for the BLADE admin tier. The Configurator inherited this logic in its own ConfigurationMonitor, so most deployments don't need a separately-deployed Watcher. It's kept around for the rare deploy that wants the pump without the Configurator UI.
