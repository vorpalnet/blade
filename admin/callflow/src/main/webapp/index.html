See the code behind the call

BLADE's signature is the lambda-based callflow: an entire SIP conversation written top-to-bottom in one method, with response handlers as nested lambdas — instead of the scattered doInvite / doResponse / doAck handlers of the traditional model. Trace a live call and the Source tab shows that code straight from the deployed apps — framework, BLADE services, and customer-built apps alike, byte-identical to what's running.
